# DEPENDENCY POLICY

## Regla General
No se permite introducir nuevas dependencias sin justificación técnica.

## Para cada nueva dependencia debe especificarse:
- Problema que resuelve
- Alternativa estándar
- Nivel de mantenimiento (última actualización)
- Comunidad activa
- Impacto en tamaño del proyecto

## Preferencias
- Librerías ampliamente adoptadas
- Buen soporte
- Documentación sólida

## Prohibido
- Paquetes abandonados
- Paquetes con bajo mantenimiento
- Librerías innecesarias para tareas simples
